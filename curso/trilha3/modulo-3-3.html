🗣️ Conversational philosophy: flow that emerges

AutoGen (now AG2) starts from a different premise: instead of defining a state graph or a structured team, theflow emerges from the conversation. Agents exchange messages, and progress happens naturally — like a team of people solving a problem together.

🧭Three paradigms compared

LangGraph

Explicit state graph. You design each transition. Maximum control, more code.

CrewAI

Team with defined roles and tasks. Structured collaboration with shared context.

AutoGen

Conversation between agents. Emergent flow. Actual code execution. Mutual reflection.

💡When the conversation wins the graph

  • •Exploratory research:the agent doesn’t know how many iterations it will need — the conversation determines
  • •Iterative gen code:generates code → executes → sees error → corrects → repeats until it works
  • •Debates and review:multiple perspectives that emerge naturally from the exchange of messages

🤝 AssistantAgent and UserProxyAgent

The fundamental pair of AutoGen:AssistantAgentuses an LLM to reason and generate code, whileUserProxyAgentcan run this code locally and simulate the user role. Together, they form a fully automatic generation-execution-feedback loop.

⚙️The basic pair in action

import autogen

# LLM configuration
config_list = [{"model": "gpt-4o", "api_key": "sk-..."}]
llm_config = {"config_list": config_list, "cache_seed": 42}

# AssistantAgent: uses LLM, generates responses and code
assistente = autogen.AssistantAgent(
    name="Cientista_de_Dados",
    llm_config=llm_config,
    system_message="""Você é um cientista de dados experiente.
    Quando solicitado, escreva código Python comentado e executável.
    Sempre encerre com 'TERMINATE' quando a tarefa estiver completa."""
)

# UserProxyAgent: executes code, simulates user
executor = autogen.UserProxyAgent(
    name="Executor",
    human_input_mode="NEVER",  # no human intervention
    max_consecutive_auto_reply=10,
    is_termination_msg=lambda x: "TERMINATE" in x.get("content", ""),
    code_execution_config={
        "work_dir": "workspace",    # execution directory
        "use_docker": False,        # True for Docker sandbox
        "timeout": 120,             # timeout in seconds
        "last_n_messages": 3        # message context
    }
)

# Start the conversation — the loop happens automatically
executor.initiate_chat(
    assistente,
    message="Analise o arquivo sales_data.csv e gere um gráfico das vendas por mês. "
            "Inclua média móvel de 3 meses."
)

# What happens automatically:
#1. Executor sends the message
# 2. Wizard generates Python code
# 3. Executor executes the code
#4. If error: returns stderr to Wizard
# 5. Wizard fixes and tries again
# 6. When successful: Wizard ends with TERMINATE

💡human_input_mode has 3 options

  • •"NEVER"— fully autonomous, without human interruption
  • •"TERMINATE"— human can intervene when the agent wants to finish
  • •"ALWAYS"— human approves each message before sending

👥 GroupChat: multiple agents, one conversation

When you need more than two agents collaborating,GroupChatis AutoGen's native engine. A GroupChatManager decides who talks next based on rules or via another LLM.

🏛️GroupChat with 4 agents

#4 Specialized agents
pesquisador = autogen.AssistantAgent(
    name="Pesquisador",
    llm_config=llm_config,
    system_message="Você pesquisa informações e coleta dados relevantes."
)

analista = autogen.AssistantAgent(
    name="Analista",
    llm_config=llm_config,
    system_message="Você analisa dados e identifica padrões e insights."
)

critico = autogen.AssistantAgent(
    name="Critico",
    llm_config=llm_config,
    system_message="""Você critica construtivamente as análises dos outros.
    Aponte falhas de lógica, dados faltantes e conclusões precipitadas."""
)

executor = autogen.UserProxyAgent(
    name="Executor",
    human_input_mode="NEVER",
    code_execution_config={"work_dir": "ws", "use_docker": False}
)

# Create GroupChat
group_chat = autogen.GroupChat(
    agents=[pesquisador, analista, critico, executor],
    messages=[],
    max_round=20,                          # maximum shifts
    speaker_selection_method="auto",        # LLM decides who speaks
    allow_repeat_speaker=False             # avoid monologue
)

# Manager coordinates the conversation
manager = autogen.GroupChatManager(
    groupchat=group_chat,
    llm_config=llm_config  # use LLM to choose speaker
)

# Start the debate
executor.initiate_chat(
    manager,
    message="Pesquise e analise o impacto do GPT-4o em startups B2B SaaS. "
            "O Crítico deve questionar cada conclusão."
)

📊Speaker selection methods

  • •"auto"— LLM decides who should speak based on the context of the conversation
  • •"round_robin"— each agent speaks in rotating order (deterministic)
  • •"random"— random agent every turn
  • •função customizada— you define the logic of the speaker when

🪞 Reflection between agents: the automatic critic

The reflection pattern uses two agents: onegeneratorand onecritical. The generator produces a response, the critic evaluates it with a clear rubric, and the cycle continues until the critic approves or the threshold is reached.

🔄Writer-critic pattern

# Writer Agent
escritor = autogen.AssistantAgent(
    name="Escritor",
    llm_config=llm_config,
    system_message="""Você escreve análises técnicas claras e precisas.
    Quando receber crítica, melhore o texto e marque com [REVISADO vX]."""
)

# Critical agent with clear rubric
critico = autogen.AssistantAgent(
    name="Critico",
    llm_config=llm_config,
    system_message="""Avalie textos com esta rubrica (1-5 cada):
    1. Precisão técnica: dados e fatos corretos?
    2. Clareza: texto acessível ao público-alvo?
    3. Completude: todos os aspectos importantes cobertos?
    4. Estrutura: organização lógica e fluente?

    Se TODOS >= 4: responda com "APROVADO: [motivo]"
    Senão: liste melhorias específicas por critério."""
)

# Controlled reflection loop
proxy = autogen.UserProxyAgent(
    name="Proxy",
    human_input_mode="NEVER",
    max_consecutive_auto_reply=8,  # maximum 4 writer+critic cycles
    is_termination_msg=lambda x: "APROVADO" in x.get("content", ""),
    code_execution_config=False
)

# Starts with the topic for the writer
proxy.initiate_chat(
    autogen.GroupChatManager(
        groupchat=autogen.GroupChat(
            agents=[escritor, critico, proxy],
            messages=[],
            speaker_selection_method="round_robin"
        ),
        llm_config=llm_config
    ),
    message="Escreva uma análise de 300 palavras sobre LangGraph para CTOs de startup."
)

💡Clear rubric = actionable feedback

The critic needs a specific rubric to give actionable feedback. "It's good" or "better" are useless. "Clarity (3/5): Paragraph 2 uses jargon without definition for the intended audience" is what the writer needs to improve.

💻 Sandbox code execution

Code execution is AutoGen's superpower. The AssistantAgent generates Python, the UserProxyAgent runs in a controlled environment, stdout/stderr becomes automatic feedback, and the agent corrects until it works — without any human intervention.

Local execution (development)

code_execution_config={
    "work_dir": "workspace",
    "use_docker": False,
    "timeout": 60
}
  • • No Docker overhead
  • • Access to local resources
  • • Risk: malicious code may execute
  • • Ideal for development and testing

Docker sandbox (production)

code_execution_config={
    "work_dir": "workspace",
    "use_docker": "python:3.11-slim",
    "timeout": 120
}
  • • Complete system isolation
  • • Code does not access external resources
  • • Container initialization overhead
  • • Mandatory in production with generated code

⚠️Security in code execution

Never run LLM-generated code in production without sandboxing. LLM may — intentionally or unintentionally — generate code that accesses sensitive files, makes unauthorized network calls, or consumes excessive resources.

  • ✗No sandbox in production
  • ✗Docker with network access without allowlist
  • ✗No timeout configured
  • ✓Docker with isolated network for production

🎯 When to prefer AutoGen over others

AutoGen is not the right answer for everything. But there are cases where it outperforms LangGraph and CrewAI: when the flow needs to emerge naturally, when actual code execution is central, or when debates between multiple perspectives are the core of the problem.

1

Iterative code generation and debugging

The agent needs to write Python, run it, see the error, fix it and repeat until it works. AutoGen was built exactly for this.

2

Exploratory research without fixed steps

When you don't know how many iterations it will take or what the next question will be — conversation sets the path.

3

Analysis with multiple perspectives

Debates, critical reviews, risk analyzes where different "voices" need to contradict each other and reach an emerging consensus.
