🏢 The concept of Crew: team with a mission

CrewAI models multi-agent systems aswork teams. A Crew has a collective mission, and each member has a specific role, objective and context. Unlike LangGraph which thinks in graphs, CrewAI thinks inorganizational structures.

🏗️CrewAI Architecture

🎭
agent
Member with role, goal and backstory
📋
Task
Assignment with description and expected_output
🔧
Tool
Capabilities: web, code, API
🚀
Crew
Orchestra agents + tasks + process

🧠The difference in mentality

With CrewAI youdon't think about data flow— do you think aboutwho does what. What are the roles? What are the responsibilities? How does information flow between team members? It's team management, not graph programming.

  • •LangGraph:"What's the next node after this one?"
  • •CrewAI:"Who has the skills to do this?"

🎭 Defining agents with roles

Each Agent is defined by three attributes that build their persona:role(title and function),goal(what you want to achieve) andbackstory(context and specialty). Together, they form the agent's system prompt.

👤Three complete agents

from crewai import Agent
from crewai_tools import SerperDevTool, WebsiteSearchTool

search_tool = SerperDevTool()

#Agent 1: Market Researcher
pesquisador = Agent(
    role="Senior Market Research Analyst",
    goal="Coletar dados precisos e atualizados sobre tendências de mercado no setor solicitado",
    backstory="""Você é um analista sênior com 10 anos de experiência em pesquisa
    de mercado. Especialista em identificar padrões emergentes, analisar dados
    qualitativos e quantitativos, e sintetizar informações complexas em insights
    acionáveis. Sua metodologia é rigorosa e suas fontes são sempre verificadas.""",
    tools=[search_tool],
    llm="gpt-4o",
    verbose=True,
    allow_delegation=False  # cannot delegate sub-tasks
)

# Agent 2: Strategic Analyst
analista = Agent(
    role="Strategic Business Analyst",
    goal="Transformar dados brutos em análises estratégicas com recomendações claras",
    backstory="""Especialista em análise SWOT, Porter's Five Forces e frameworks
    estratégicos. Você transforma dados e pesquisas em narrativas coerentes que
    executivos conseguem usar para tomar decisões. Seu diferencial é conectar
    pontos que outros não veem.""",
    llm="gpt-4o",
    verbose=True
)

#Agent 3: Executive Writer
redator = Agent(
    role="Executive Content Writer",
    goal="Criar relatórios executivos claros, concisos e persuasivos a partir de análises",
    backstory="""Redator especializado em comunicação executiva. Você sabe como
    apresentar dados complexos de forma que C-level entenda em 5 minutos.
    Usa estrutura clara: executive summary, principais achados, recomendações
    e próximos passos.""",
    llm="gpt-4o-mini",  # essay uses smaller template
    verbose=True
)

💡The backstory makes all the difference

A generic backstory generates generic answers. A specific backstory with years of experience, favorite methodologies and clear differentiators makes the LLM truly “come to fruition”. Invest time in writing rich backstories — it's your equivalent of prompt engineering for agent personas.

📋 Tasks and flows: structuring work

A Task is a unit of work with three mandatory elements:description(what should be done),expected_output(what format and content is expected) andagent(who executes). The expected_output of course is the secret to consistent outputs.

📝Tasks complete with context

from crewai import Task
from pydantic import BaseModel
from typing import List

# Final output schema
class RelatorioMercado(BaseModel):
    setor: str
    principais_tendencias: List[str]
    oportunidades: List[str]
    ameacas: List[str]
    recomendacoes: List[str]
    score_atratividade: float

# Task 1: Research
task_pesquisa = Task(
    description="""Pesquise o mercado de {setor} focando em:
    1. Tamanho de mercado atual e projeções 2025-2027
    2. Top 5 players e suas estratégias
    3. Tendências tecnológicas emergentes
    4. Barreiras de entrada
    5. Comportamento do consumidor""",
    expected_output="""Relatório de pesquisa com dados quantitativos
    de pelo menos 5 fontes verificadas. Include: tamanho de mercado (USD),
    CAGR, principais players com market share, e 3 tendências com evidências.""",
    agent=pesquisador,
    output_file="pesquisa_raw.md"  # save to file
)

# Task 2: Analysis (uses research output as context)
task_analise = Task(
    description="""Com base na pesquisa de mercado fornecida, realize:
    1. Análise SWOT completa do setor
    2. Mapeamento de oportunidades não óbvias
    3. Avaliação de riscos com probabilidade e impacto
    4. Score de atratividade do mercado (0-10)""",
    expected_output="""Análise estratégica estruturada com SWOT,
    mapa de oportunidades rankeadas por potencial, matriz de risco
    e score de atratividade com justificativa.""",
    agent=analista,
    context=[task_pesquisa]  # receives output from the previous task
)

# Task 3: Writing (output in Pydantic)
task_relatorio = Task(
    description="""Crie um relatório executivo baseado na pesquisa e análise.
    O relatório deve ser legível em 5 minutos por um CEO.""",
    expected_output="Relatório executivo em formato JSON estruturado.",
    agent=redator,
    context=[task_pesquisa, task_analise],
    output_pydantic=RelatorioMercado  # typed output!
)

🏛️ Agent hierarchy: the manager pattern

No Process.hierarchical, an LLM-powered Manager Agent coordinates the others — decides who performs each task, can redirect work and review outputs before continuing. It's like having an automatic project manager.

Process.sequential (default)

Tasks execute in a fixed order. Output of one becomes input of the next automatically.

  • ✓Simple and predictable
  • ✓Easy to debug
  • ✓Less token cost
  • ✗No dynamic adaptation

Process.hierarchical

LLM Manager decides who does what. Can dynamically redirect, review, and coordinate.

  • ✓Adapting to unexpected situations
  • ✓Better output quality
  • ✗More tokens, more cost
  • ✗Less deterministic

👔Manager agent in code

from crewai import Crew, Process

# Manager agent (uses strong model for coordination)
manager = Agent(
    role="Research Project Manager",
    goal="Garantir que a pesquisa de mercado seja completa, precisa e entregue no prazo",
    backstory="Gerente de projetos com experiência em coordenar times de análise.",
    llm="gpt-4o",
    allow_delegation=True  # MUST have active delegation
)

# Crew with hierarchy
crew_hierarquica = Crew(
    agents=[pesquisador, analista, redator],
    tasks=[task_pesquisa, task_analise, task_relatorio],
    process=Process.hierarchical,
    manager_agent=manager,
    verbose=True,
    memory=True  # shared memory between agents
)

resultado = crew_hierarquica.kickoff(
    inputs={"setor": "IA Generativa B2B"}
)

# Access typed output
relatorio: RelatorioMercado = resultado.pydantic
print(f"Score de atratividade: {relatorio.score_atratividade}")
print(f"Recomendações: {relatorio.recomendacoes}")

⚡ Sequential vs. sequential process parallel

Tasks that don't depend on each other waste time if they are sequential. Withasync_execution=True, they run in parallel — being able to reduce latency from minutes to seconds.

⚡Parallelism in independent tasks

# Scenario: research 3 markets simultaneously
task_mercado_br = Task(
    description="Analise o mercado de IA no Brasil",
    expected_output="Relatório mercado Brasil",
    agent=pesquisador,
    async_execution=True  # runs in parallel!
)

task_mercado_us = Task(
    description="Analise o mercado de IA nos EUA",
    expected_output="Relatório mercado EUA",
    agent=pesquisador,
    async_execution=True  # also parallel!
)

task_mercado_eu = Task(
    description="Analise o mercado de IA na Europa",
    expected_output="Relatório mercado Europa",
    agent=pesquisador,
    async_execution=True  # also parallel!
)

# This task waits for all previous ones to finish
task_comparativo = Task(
    description="Compare os três mercados e identifique o mais atrativo",
    expected_output="Comparativo global com ranking",
    agent=analista,
    context=[task_mercado_br, task_mercado_us, task_mercado_eu],
    # no async_execution — wait for context
)

# Crew executes: 3 searches in parallel, then the comparison
crew = Crew(
    agents=[pesquisador, analista],
    tasks=[task_mercado_br, task_mercado_us, task_mercado_eu, task_comparativo],
    process=Process.sequential
)

💡Golden rule of parallelism

Useasync_execution=Truein any task that does not need the output of another task. If A, B, and C are independent, but D needs all three — A, B, C are async; D is synchronous with context=[A, B, C]. This can reduce latency by 3x.

📈 Project: market analysis team

The complete project: three agents (researcher, analyst, writer), three tasks in sequence, automatic context sharing and final output typed in Pydantic. This is the CrewAI standard for enterprise use.

🚀Complete crew in production

# Complete market analysis crew
from crewai import Crew, Process
from datetime import datetime

crew_analise = Crew(
    agents=[pesquisador, analista, redator],
    tasks=[task_pesquisa, task_analise, task_relatorio],
    process=Process.sequential,
    verbose=True,
    memory=True,           # memory between tasks
    max_rpm=10,            # API rate limiting
    output_log_file=f"log_{datetime.now():%Y%m%d_%H%M}.txt"
)

# Run with dynamic inputs
resultado = crew_analise.kickoff(inputs={
    "setor": "SaaS de Recursos Humanos",
    "regiao": "América Latina",
    "foco": "PMEs com 50-500 funcionários"
})

# Structured and typed output
relatorio = resultado.pydantic
print(f"Setor analisado: {relatorio.setor}")
print(f"Score de atratividade: {relatorio.score_atratividade}/10")
print("\nPrincipais tendências:")
for i, tendencia in enumerate(relatorio.principais_tendencias, 1):
    print(f"  {i}. {tendencia}")
print("\nRecomendações estratégicas:")
for rec in relatorio.recomendacoes:
    print(f"  → {rec}")

# Execution costs and metrics
print(f"\nTokens usados: {resultado.token_usage}")
print(f"Tempo total: {resultado.tasks_output[-1].raw[:100]}...")

CrewAI shines when...

  • ✓Teams of agents with clear roles
  • ✓Rapid workflow prototyping
  • ✓Final structured output (Pydantic)
  • ✓Automatic delegation logic

Limitations of CrewAI

  • ✗Flow control less accurate than LangGraph
  • ✗Complex loops are more difficult to implement
  • ✗Observability requires additional configuration
  • ✗Less control over internal state
