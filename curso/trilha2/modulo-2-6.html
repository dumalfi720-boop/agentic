🔌 What is MCP

Model Context Protocol (MCP) is theAnthropic open standardto connect agents to external tools in a standardized, plug-and-play way. Just as USB eliminated the chaos of proprietary connectors, MCP standardizes how agents connect to any tool.

🔌 The USB analogy is accurate

Before MCP (no standard)

  • ✗Each agent implements custom integration for each tool
  • ✗Slack Tool for Claude ≠ Slack Tool for GPT-4
  • ✗Rewrite integrations when switching LLM

With MCP

  • ✓Slack's MCP Server works with any MCP client
  • ✓Switch LLM without rewriting integrations
  • ✓Ready-to-use MCP server ecosystem

📊 Adoption in 2026

  • •Claude Code, Claude Desktop and Claude API support MCP natively
  • •Cursor, Windsurf and other AI-first IDEs have adopted the protocol
  • •Official MCP-servers repository has 100+ ready-made servers (GitHub, Postgres, Slack, Jira...)
  • •OpenAI announced support for the protocol — becoming an industry standard

🏗️ Server/client architecture

The MCP architecture has two roles: theMCP Serverexposes tools (functionalities) via standardized protocol, and theMCP Client(your agent) discovers and calls these tools. Communication can be local (stdio) or remote (HTTP).

Complete architecture

MCP Client

Your agent (Claude, LangChain, any framework)

Automatically discovers tools at startup

JSON-RPC 2.0

← →

stdio or HTTP+SSE

MCP Server

Exposes tools, resources and prompts via protocol

It could be Postgres, Slack, GitHub, your API...

Transport: studio

The server runs as a child process of the client. Communication via stdin/stdout. Ideal for local servers (same computer).

# config on the client

{"command": "python", "args": ["my_server.py"]}

Transport: HTTP + SSE

The server runs as an independent web service. Client connects via HTTP. Ideal for production, multiple simultaneous clients.

# config on the client

{"url": "https://my-server.com/mcp"}

🗃️ Connecting database via MCP

Using the official Postgres MCP server, you can connect any agent to your database in minutes. The agent becomes empoweredexecute queries, list tables and analyze data in natural language— without any extra code in the agent.

Configuration in 3 steps

1

Install the Postgres MCP server

npm install -g @modelcontextprotocol/server-postgres

2

Configure in the client settings file

{"mcpServers": {"postgres": {

"command": "mcp-server-postgres",

"args": ["postgresql://user:pass@localhost/mydb"]

}}}

3

Done — the agent can now interact with the bank

User: "What are the 10 customers with the highest outstanding value?" → Agent calls query_postgres automatically.

⚠️ Mandatory security in bank MCP

  • •Use database user with READ-ONLY (SELECT) permission only
  • •Never expose connection string with admin user
  • •Limit visible tables via restricted schema
  • •Log all queries executed by the agent

🌐 Connecting external APIs

Any REST API can be encapsulated as an MCP server. You define the tools (endpoints) with your schemas and the agent starts tocall the API naturally, as if it were an internal function — without integration code in the agent.

Example: REST API wrapper as MCP server

from mcp.server import Server

from mcp.server.models import InitializationOptions

import httpx

server = Server("my-api-crm")

@server.tool()

async def fetch_cliente(cliente_id: str) -> str:

"""Search complete customer data by ID in CRM."""

async with httpx.AsyncClient() as client:

response = await client.get(

f"https://api.meucrm.com/customers/{cliente_id}",

headers={"Authorization": f"Bearer {API_KEY}"}

)

return response.json()

# The schema is automatically generated from type hints and docstring!

Popular use cases

  • • CRM (Salesforce, HubSpot)
  • • Ticket system (Jira, Linear)
  • • Email (Gmail API)
  • • Calendar (Google Calendar)

Benefits of the MCP approach

  • • Written once, used by multiple agents
  • • Auth centralized on the MCP server
  • • Versioned independent of the agent
  • • Testable without the agent

OpenAPI → MCP

  • • Tools that convert OpenAPI spec to MCP server automatically
  • • Reduces manual implementation for APIs with swagger

🤖 MCP on Claude Code

Claude Code uses MCP natively. When adding an MCP server to the configuration, Claude Codeautomatically discovers available toolsand starts using them in its responses — without any modifications to the agent's code.

Configuration in claude_desktop_config.json

// ~/.config/claude/claude_desktop_config.json

{

"mcpServers": {

"postgres": {

"command": "mcp-server-postgres",

"args": ["postgresql://readonly:senha@localhost/prod"]

},

"github": {

"command": "mcp-server-github",

"env": {"GITHUB_TOKEN": "ghp_..."}

},

"my-crm": {

"command": "python",

"args": ["path/to/my_crm_server.py"]

}

}

}

💡 How Claude Code uses MCP tools

After adding the servers, restart Claude Code. It automatically lists available tools and uses them when relevant. Example: with the GitHub server configured, you can ask "Open a PR with these changes" and Claude Code will create the PR directly via MCP — without you copying and pasting code.

🏗️ Creating your first MCP server

With the official MCP Python SDK, you can create a working server in less than30 lines. Defines tools with decorator, implements logic and registers on the client. Simple as creating a Python function.

Minimum full MCP server

#!/usr/bin/env python3

"""pip install mcp"""

import asyncio

from mcp.server import Server

from mcp.server.stdio import stdio_server

server = Server("my-first-server")

@server.tool()

async def greeting(name: str) -> str:

"""Generates a personalized greeting for the given name.

Use when you need to formally greet someone."""

return f"Hello, {name}! Welcome to the agent system."

@server.tool()

async def calculate_discount(price: float, discount_percent: float) -> str:

"""Calculates the final price after applying a percentage discount."""

final_price = price * (1 - discount_percent / 100)

return f"Price with {discount_percent}% discount: R$ {final_price:.2f}"

if __name__ == "__main__":

asyncio.run(stdio_server(server))

# Run: python meu_server.py

# Test: npx @modelcontextprotocol/inspector python meu_server.py

What the SDK automatically generates

  • ✓JSON Schema of the tool from Python type hints
  • ✓Description of the tool from the docstring
  • ✓Endpoint discovery tool for the customer
  • ✓Parameter serialization/deserialization

Testing with MCP Inspector

MCP Inspector is a UI for testing your MCP servers without needing an agent:

npx @modelcontextprotocol/inspector

python my_server.py

Open web UI to call tools manually and see results
