========================================
WHAT IS ANTIGRAVITY?
========================================

Antigravity is a code editor created by Google.
How it differs from normal editors (like VS Code)
and that it was built from scratch to work with a
artificial intelligence agent - Gemini.

In a normal IDE, you write all the code. The Gemini
Maybe suggest a line here or there, but you get the job done.

In Antigravity, the logic is different. You describe what
want to do it in normal language - like sending a message -
and the agent plans and writes the code for you. You review
the plan, approves it, monitors its execution and verifies the result.

For example, instead of spending 2 hours writing a system
login, you write:

    "Add login with Google. The user must have an email
    and display name saved in the database. After login, redirect
    to the home page."

The agent reads this instruction, creates a plan with all the files
that will change, waits for you to approve it and then writes the code.

This is called "agentic development" - you direct the
project, the agent writes.

WHO CAN USE ANTIGRAVITY?
========================================

To use Antigravity you need:

1. Have a Google account (Gmail will do)

2. Have access to the Antigravity beta program
   Antigravity is not yet available to everyone.
   And I need to request access.

3. Have Node.js installed on your computer
   Node.js is a program that allows you to run JavaScript code
   outside the browser. And free.

4. Have Git installed
   Git is a program for code version control.
   It's also free.

If you only have a Google account, start with the next topic.
The instructions show you how to get everything you need.


========================================
HOW TO GET ACCESS TO ANTIGRAVITY
========================================

Step 1: Access the beta program website
-----------------------------------------
Open your browser and go to:
    https://g.co/antigravity

You will need to be logged into your Google account.

Step 2: Request access
---------------------------
Click the "Request Access" button.
A simple form will appear.

In the form, briefly describe how you want to use
o Antigravity. You can write in English or Portuguese.
Example:
    "I want to use it to learn agentic development
    and automate repetitive programming tasks."

Click submit.

Step 3: Wait for the email
-------------------------
Google will review your request. This can take
from a few hours to a few days. You will receive an email
in Gmail when access is granted.

While waiting, install the programs in the next topic.

Step 4: Install Antigravity
-------------------------------
When you receive the approval email, click on the approval link
download that turns into the email. Download the installer for
your operating system (Windows, macOS or Linux).

Run the installer. The first time you open the
Antigravity, he will ask you to enter the
your Google account. Use the same account with which
requested access.


========================================
HOW TO INSTALL PREREQUISITES
========================================

You need to install two programs before using
the files for this project:

Program 1: Node.js
--------------------
Node.js is required for MCP servers, which are
tools the agent uses to access your files
and GitHub.

To install:
    1. Visit: https://nodejs.org
    2. Click the big green button that says "LTS"
       (this downloads the recommended stable version)
    3. Open the downloaded file and follow the instructions
    4. To check if it worked, open the terminal and type:
           node --version
       It should say something like "v20.11.0"

Program 2: Git
----------------
Git is necessary to save versions of your code and
for the agent to create commits and branches.

To install:
    1. Visit: https://git-scm.com
    2. Click "Download" and download to your system
    3. Install following the on-screen instructions
4. To check, open the terminal and type:
           git --version
       It should say something like "git version 2.44.0"


========================================
HOW TO USE THIS PROJECT FILES
========================================

This project contains two main files that you
will use within Antigravity:

File 1: workspace.json
--------------------------
This file tells Antigravity how to behave in
your project. It defines which AI model to use, which
enable tools and what the agent needs to ask
permission to do.

How to use:
    1. Open Antigravity
    2. Create a new project pointing to the project folder
       your code (or create a new folder)
    3. Copy the workspace.json file to the project root
    4. Antigravity will automatically detect and load

File 2: CONTEXT.md
----------------------
This is the most important file. It's like a manual
of your project that the agent reads before starting any
task. You need to customize this file for the
YOUR project.

How to use:
    1. Copy the CONTEXT.md file to the root of your project
    2. Open the file in any text editor
    3. Replace the example project information with the
       real information about your project:
       - Project name
       - What the project does
       - What technologies do you use
       - Your code conventions
    4. Save the file

If your project is simple, use the file at:
    examples/context-minimal.md
It has fewer sections and is easier to fill out.

Installing the extra tools (MCP servers):
---------------------------------------------------
So that the agent can access your files and GitHub,
you need to install two extra programs. We have a
script that does this automatically.

Open the terminal in the project folder and run:
    chmod +x scripts/setup-mcp.sh
    ./scripts/setup-mcp.sh

The script will install everything and show colorful messages
indicating what is happening. If everything goes well,
You will see a green success message at the end.


========================================
HOW TO USE ANTIGRAVITY IN EVERYDAY LIFE
========================================

With everything configured, using Antigravity is simple.

Opening a project:
-------------------
    1. Open Antigravity
    2. Go to File > Open Folder
    3. Select your project folder

The agent will load workspace.json and CONTEXT.md
automatically. He now "knows" his project.

Asking the agent to do something:
-----------------------------------
At the bottom of the Antigravity screen there is a field
chat. This is where you communicate with the agent.

To implement a new feature, write:

    @agent implement feature: [describe what you want]

Example:
    @agent implement feature: add profile page
    username with editable photo, name and bio

To fix a bug:
    @agent fix bug: [describe the problem]

Example:
    @agent fix bug: the save button on the save page
    settings does nothing when clicked

To deploy (publish the website):
    @agent deploy to staging

Reviewing what the agent will do:
------------------------------------
After receiving your order, the agent FIRST shows
a plan of what you intend to do. Read this plan first
to approve.

The plan lists all files that will be created or
changed. If something seems wrong, write your correction
in chat before approving.

To approve the plan, click "Approve" or write "s".

Monitoring the execution:
-------------------------
After approval, the agent writes the code. Do you see
files being created in real time on the screen
Antigravity. You don't need to do anything - just wait.

If the agent encounters an error (for example, a test
that doesn't pass), it will try to correct it automatically.

At the end, the agent presents a summary of what was done
and waits for your final approval before committing the code.


========================================
WHAT TO EXPECT AS A RESULT
========================================

When using Antigravity with the files in this project,
you should expect:

What will work well:
    - Implement new features following the guidelines
      patterns of your project
    - Fix bugs with clear and well-described cause
    - Create automated tests for existing code
    - Refactor code to improve quality
    - Deploy to test environments (staging)

What requires more attention:
- Very complex functionalities may require
      multiple review cycles
    - The agent may not know very new libraries
      or very specific
    - Important architectural decisions still need to be made
      from you - the agent is good at executing, not
      decide product direction

How much time you save:
    For repetitive and well-defined tasks, the agent
    can be 3 to 5 times faster than writing
    the code manually. For creative tasks or
    exploration companies, the gain is lower - the value is more
    in helping you think than in writing speed.


========================================
COMMON QUESTIONS
========================================

Question: Do I need to know how to program to use Antigravity?
Answer: Yes. Antigravity accelerates development,
but it does not replace technical knowledge. Do you need
understand the code that the agent generates to review and
approve of what he does. Without this, you won't know if
the result is correct.

Question: Can the agent delete my code by accident?
Answer: With the settings of this project (workspace.json),
the agent always asks for permission before operations
destructive. Furthermore, with Git configured, you can
You can undo any change with one command.

Question: Is my code sent to Google when I use Antigravity?
Answer: Yes. The code that the agent accesses is sent
to Google servers to process requests
from Gemini. Check Antigravity terms of service
before using with projects containing sensitive data
or restricted proprietary code.

Question: What happens if the agent makes a mistake?
Answer: You review the plan before execution. If the
agent makes an error in execution, you can describe
the problem in the chat and he will try to fix it. For errors
serious, Git allows you to return to the previous state of the code.
