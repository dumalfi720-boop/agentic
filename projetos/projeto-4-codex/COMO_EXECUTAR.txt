------------------------------------------------------------
WHAT IS CODEX CLI?
------------------------------------------------------------

The Codex CLI is a command line tool created
by OpenAI that allows you to use artificial intelligence
directly in your software project.

In simple language: it is a programming assistant that
"see" your code and can help:

  - Explain how the code works
  - Find bugs and suggest fixes
  - Automatically write tests
  - Refactor code (improve without changing what it does)
  - Create new files following project standards
  - Review Pull Requests before merging

The difference to other AI assistants (like ChatGPT
on the web) and that the Codex CLI ACCESSES your files directly.
It sees the actual code, not a copy you pasted.

------------------------------------------------------------
STEP 1 — OPEN THE TERMINAL
------------------------------------------------------------

The terminal (also called the "command prompt" or
"console") and where you enter text commands.

On Linux:
  - Press Ctrl+Alt+T
  - Or search for "Terminal" in the application menu

On macOS:
  - Press Cmd+Space and type "Terminal"
  - Or go to Applications > Utilities > Terminal

On Windows:
  - Press Win+R, type "cmd" and press Enter
  - Or search for "PowerShell" in the Start menu
  - RECOMMENDED: Install Windows Terminal from the Microsoft Store

------------------------------------------------------------
STEP 2 — INSTALL NODE.JS
------------------------------------------------------------

The Codex CLI needs Node.js to work. Node.js and
a program that allows you to run JavaScript outside the browser.

To check if it is already installed, open the terminal and
type (then press Enter):

  node --version

If something like "v20.0.0" appears, skip to Step 3.

If an error appears, install Node.js:

  On Linux (Ubuntu/Debian):
    Copy and paste this command into the terminal:

    curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

    Then close and reopen the terminal, and run:

    nvm install 20
    nvm use 20

  On macOS:
    Visit https://nodejs.org
    Download the "LTS" installer (Long Support Version)
    Run the installer like any other program

  On Windows:
    Visit https://nodejs.org
    Download the ".msi" installer for Windows
    Run and follow the instructions (Next, Next, Install)

Check the installation:
  node --version
  (Must appear v18.0.0 or higher)

------------------------------------------------------------
STEP 3 — INSTALL CODEX CLI
------------------------------------------------------------

With Node.js installed, install the Codex CLI.
Type in the terminal:

  npm install -g @openai/codex

The "-g" stands for "global" — the Codex will be available in
anywhere on your computer.

To check if you installed correctly:

  codex --version

------------------------------------------------------------
STEP 4 — GET THE OPENAI API KEY
------------------------------------------------------------

To use Codex, you need an OpenAI account and
of an "API key" (access key).

1. Visit: https://platform.openai.com

2. Create an account (or log in if you already have one)

3. In the side menu, click on "API Keys"
   (or access directly: https://platform.openai.com/api-keys)

4. Click on "Create new secret key"

5. Enter a name to identify it (ex: "my-computer")

6. Click on "Create secret key"

7. IMPORTANT: Copy the key that appears!
   It starts with "sk-" and only appears ONCE.
   If you close without copying, you will need to create another one.

The key looks like this:
  sk-proj-AbCdEfGhIjKlMnOpQrStUvWxYz123456...

WARNING: Treat your API key like a password. Never share
with anyone or put it in public places (GitHub, etc.).

COST NOTICE: Use of the OpenAI API is pay-per-use.
For casual use and studies, the cost is low (cents per
hour of use). You can set spending limits on:
  https://platform.openai.com/settings/organization/limits
------------------------------------------------------------
STEP 5 — CONFIGURE THE API KEY
------------------------------------------------------------

Now let's tell Codex what your key is.

On Linux and macOS:
  Type in the terminal (replace sk-... with your real key):

  echo 'export OPENAI_API_KEY="sk-SUA_CHAVE_AQUI"' >> ~/.bashrc
  source ~/.bashrc

  If using zsh (default on modern macOS):

  echo 'export OPENAI_API_KEY="sk-SUA_CHAVE_AQUI"' >> ~/.zshrc
  source ~/.zshrc

On Windows (PowerShell):
  [System.Environment]::SetEnvironmentVariable("OPENAI_API_KEY", "sk-SUA_CHAVE_AQUI", "User")

To check if it worked:
  Linux/macOS: echo $OPENAI_API_KEY
  Windows: echo %OPENAI_API_KEY%

If your key (or part of it) appears, it is set!

------------------------------------------------------------
STEP 6 — TURN THE FIRST CONTROLS
------------------------------------------------------------

Navigate to the project directory:

  cd /home/nmaldaner/projetos/agentic/projetos/projeto-4-codex

Start Codex in interactive mode:

  codex

Codex will open a prompt where you can type
questions and instructions. Try these examples:

  "What files are in this project?"

  "Explain what the AGENTS.md file is"

  "What tools do I need to work with this?"

To exit the interactive Codex, press Ctrl+C.

------------------------------------------------------------
EXAMPLES OF USEFUL COMMANDS
------------------------------------------------------------

Review mode (only reads, doesn't change anything):

  codex --approval-mode suggest "list the points for improvement in the code"

Edit mode (can create/edit files):

  codex --approval-mode auto-edit "add comments to file X"

Run without interaction (for automation):

  bash scripts/run-headless.sh "your instruction here"

Automatic environment setup:

  bash scripts/setup.sh

------------------------------------------------------------
WHAT TO DO IF SOMETHING GOES WRONG
------------------------------------------------------------

PROBLEM: "command not found: codex"
SOLUTION: Codex was not installed or the terminal was not
         restarted. Try closing and reopening the terminal,
         and then run: npm install -g @openai/codex

PROBLEM: "Error: OPENAI_API_KEY is not set"
SOLUTION: The API key is not configured. Follow Step 5
         again and make sure to run "source ~/.bashrc"

PROBLEM: "Error: Insufficient quota"
SOLUTION: Your OpenAI account has no credit. Access
         https://platform.openai.com/settings/billing
         and add a payment method.

PROBLEM: "Error: Rate limit exceeded"
SOLUTION: You made too many requests quickly. Wait
         a few seconds and try again.

PROBLEM: "node: command not found"
SOLUTION: Node.js is not installed or is not in the PATH.
         Repeat Step 2.

PROBLEM: "Permission denied" when installing
SOLUTION: On Linux/macOS, try:
         sudo npm install -g @openai/codex
         (it will ask for your computer password)

PROBLEM: Codex does not understand the project
SOLUTION: Open the AGENTS.md file and add more context
         about your project. The more detailed, the better.
