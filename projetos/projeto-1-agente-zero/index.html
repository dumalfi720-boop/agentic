── TOPIC 2: Defining Tools ────────────────────

What is it

Tools (tools) are external capabilities that the language model can invoke. Each tool is described by a JSON schema that specifies name, description and parameters. Claude decideswhen and howcall each tool based on the context of the conversation.

Why learn

Tools transform a passive LLM into an active agent. Without them, the model can only generate text. With tools, it can fetch data in real time, execute code, call external APIs and persist information — making it genuinely useful.

Key concepts

input_schema

JSON Schema that defines the parameters accepted by the tool. Claude automatically generates valid inputs.

description

Natural language text that explains to the modelwhenuse the tool. The clearer it is, the better the decision.

tool_use block

When Claude decides to use a tool, he returns a blocktool_usewith id, name and inputs.

Schema example

tools = [

}

── TOPIC 3: The Dispatcher ──────────────────────────────

What is it

The dispatcher is the function responsible for receiving the name of the tool requested by the model and route execution to the correct Python function. It acts as an intermediary between Claude's decisions and the actual implementations of the tools.

Why learn

Without a dispatcher, the agentic loop would be chaotic. Centralize routing logic makes it easier to add new tools, handle errors uniformly, and record (log) all calls for debugging and auditing.

Key concepts

Routing by name

The dispatcher usesif/elifor a function dictionary to map tool names to implementations.

Error handling

Always return an error descriptive string — the model will use this information to correct the call in the next iteration.

safe eval()

Useeval(expr, {"__builtins__": {}})to block dangerous functions in evaluating mathematical expressions.

── TOPIC 4: The Agentic Loop ────────────────────────────

What is it

The agentic loop is the heart of the agent. It repeats the cycleThink → Act → Observeuntil the model decide you have enough information to respond to the user without calling up more tools. The termination sign is the fieldstop_reason == "end_turn".

Why learn

Understanding the manual loop reveals what frameworks like LangChain and LlamaIndex do in behind the scenes. With this knowledge, you can debug unexpected behavior, optimize the number of API calls and implement custom logic that generic frameworks do not support.

Key concepts

stop_reason

"end_turn"= final answer."tool_use"= model wants to run tools.

Message history

Each iteration adds the wizard response and tool results to the arraymensagens.

max_iterations

Security limit to avoid infinite loops and unexpected API costs.

Loop flow

whileiteration<max_iterations:
# 1. THINK — LLM processes context
response=client.messages.create(...)
# 2. Check if you are finished
ifresponse.stop_reason== "end_turn":
returnfinal_response# Exit the loop
# 3. ACT — Execute tools
forblockinresponse.content:
ifblock.type == "tool_use":
result=execute_tool(...)
# 4. OBSERVE — Adds to history
messages.append({"role": "assistant", ...})
messages.append({"role": "user", ...}) # results
── TOPIC 5: Adding Memory ──────────────────────

What is it

Memory is the agent's ability to retain and access information between tool calls. In the project, we use a simple Python dictionary (notas = {}) as working memory. In production this could be a vector database for RAG.

Why learn

An agent with no memory starts over from scratch every session. With memory he can learn user preferences, accumulate search results, and perform complex tasks of multiple steps that go beyond the context window of a single conversation.

Key concepts

Short-term memory

The arraymensagensof the current session — everything the model has seen in this conversation.

Long-term memory

The dictionarynotaspersists between calls within the same process execution.

RAG (Retrieval Augmented Generation)

Technique of searching for relevant documents in a knowledge base and injecting context before calling the model.

What is it

The execution and testing phase validates that the agent behaves as expected in different scenarios. Includes testing single tool calls, chaining multiple tools and error situations where the model needs to recover.

Why learn

Agents have non-deterministic behavior. Testing with different cases reveals when the model chooses the wrong tools, gets into loops, or fails silently. Good testing identifies these issues before production.

"""
Agent from Zero — Agentic Engineering Masterclass
duclub — Project 1: Agent without framework

Demonstrates manual agentic looping with Anthropic API:
- Tools (function calling)
- Loop Think → Act → Observe
- No external frameworks
"""
import os
import json
import anthropic
from dotenv import load_dotenv

load_dotenv()
client = anthropic.Anthropic(api_key=os.getenv("ANTHROPIC_API_KEY"))

# ─── TOOLS ─────────────────────────────── ───────────────────────────────

tools = [
    {
        "name": "search_on_web",
        "description": "Search for information on the web on any topic",
        "input_schema": {
            "type": "object",
            "properties": {
                "query": {
                    "type": "string",
                    "description": "What to look for. Be specific."
                }
            },
            "required": ["query"]
        }
    },
    {
        "name": "calculate",
        "description": "Performs mathematical calculations. Receives valid Python expression.",
        "input_schema": {
            "type": "object",
            "properties": {
                "expression": {
                    "type": "string",
                    "description": "Mathematical expression. Ex: '2 + 2' or '100 * 0.15'"
                }
            },
            "required": ["expression"]
        }
    },
    {
        "name": "save_note",
        "description": "Saves an important note or result in memory",
        "input_schema": {
            "type": "object",
            "properties": {
                "title": {"type": "string"},
                "content": {"type": "string"}
            },
            "required": ["title", "content"]
        }
    }
]

# ─── DISPATCHER ─────────────────────────────── ────────────────────────────────

notas = {}  # Simple dictionary memory

def run_tool(nome: str, inputs: dict) -> str:
    """Dispatches the correct tool and returns the result."""

    if nome == "search_on_web":
        query = inputs["query"]

# ─── LOOP AGENTIC ────────────────────────────── ───────────────────────────────

def execute_agent(mensagem_usuario: str, max_iteracoes: int = 10) -> str:
    """
    Main agentic loop:
    1. Send a message to LLM
    2. If LLM calls tools → executes and returns results
    3. Repeat until stop_reason == end_turn
    """
    mensagens = [{"role": "user", "content": mensagem_usuario}]
    iteracao = 0

    print(f"\n{'='*60}")
    print(f"AGENT INITIATED")
    print(f"{'='*60}")

    while iteracao < max_iteracoes:
        iteracao += 1
        print(f"\n[Iteration {iteration}] Thinking...")

        resposta = client.messages.create(
            model="claude-opus-4-6",
            max_tokens=4096,
            system="You are an intelligent assistant with access to tools.",
            tools=tools,
            messages=mensagens
        )

        if resposta.stop_reason == "end_turn":
            resposta_final = next(
                (b.text for b in resposta.content if hasattr(b, "text")),
                "No response."

            print(f"FINAL ANSWER:")

            return resposta_final
